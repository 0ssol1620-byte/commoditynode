## 1. Objective

Design CommodityNode so that a real visitor can move naturally from:
- discovering a live commodity move
- understanding why it matters
- seeing their next decision step
- experiencing enough product utility to trust the workflow
- hitting a clear Pro wall at the moment of highest intent
- completing a paid subscription with minimal friction

This document is not a generic marketing plan.
It is a conversion architecture for how visitors should move through the site to produce revenue.

---

## 2. Current Diagnosis

CommodityNode already has strong product ingredients:
- live price context
- hot signal reports
- commodity hubs
- forecast layer
- simulator
- pricing page

The gap is not lack of value.
The gap is conversion choreography.

Today the product often works like this:
- visitor finds a page
- visitor reads interesting analysis
- visitor may click to another page
- visitor may or may not understand why Pro is necessary now

That creates information satisfaction without enough purchase momentum.

### Core problem
Too many visitors can say:
- “This is useful.”
without immediately saying:
- “I need Pro right now because otherwise I cannot finish my decision workflow.”

### Strategic goal
Move the site from an information property to a controlled decision funnel.

---

## 3. Master Revenue Thesis

CommodityNode should monetize at the exact point where a user needs one of these three things:

1. Forward-looking confidence
- forecast agreement, ranges, and model comparison

2. Position-level translation
- which stock / sector / portfolio exposure changes now

3. Scenario certainty
- what happens if the move extends, reverses, or gets catalyzed by an event

This means Pro should not feel like “more features.”
It should feel like:
- the moment the visitor can complete the workflow instead of stopping halfway

Recommended commercial promise:
- Free helps you understand the move.
- Pro helps you decide what to do with it.

---

## 4. Primary Conversion Jobs

The user is not buying “signal reports.”
They are buying completion of one of these jobs:

### Job A: “Tell me what this commodity move means for my names.”
Best path:
- report → commodity hub → company sensitivity / related names → Pro

### Job B: “Tell me whether this move is likely to continue.”
Best path:
- homepage mover / report → forecast panel → model agreement → Pro

### Job C: “Let me stress-test this before the event hits.”
Best path:
- report / homepage / hub → simulator → company sensitivity / deeper scenarios → Pro

### Job D: “Help me build a repeatable workflow, not just read one article.”
Best path:
- archive / homepage / hub → repeated pattern exposure → pricing → Pro

## 6. Target Conversion Paths

### Path A: Search intent → report → Pro
Best for:
- organic traffic from “why up/down today” searches

Flow:
1. Google search lands on a signal report
2. Above-the-fold quick answer satisfies immediate intent
3. “Best next step” sends user to the live commodity hub
4. Hub shows price + forecast + latest report + simulator link
5. User tries to go deeper into forecast certainty or stock-level sensitivity
6. Pro wall appears exactly at the need point
7. Pricing page reinforces the exact workflow they are already using
8. Checkout

Critical rule:
Do not ask for payment before the user sees the workflow working.
Do ask for payment the moment the workflow would otherwise stop.

### Path B: Homepage → actionable setup → simulator → pricing → checkout
Best for:
- direct visitors
- social visitors
- referral traffic

Flow:
1. Homepage shows “Today’s 3 actionable setups”
2. User clicks the setup closest to their interest
3. Setup page opens report or hub
4. Strong CTA: “Run this move in the simulator”
5. Simulator shows sector translation and partial company view
6. Paid unlock positioned as: complete the scenario → get stock-level decision output
7. Pricing page mirrors that exact scenario use case
8. Checkout

### Path C: Hub → forecast → archive → Pro
Best for:
- research-minded users
- analysts
- repeat visitors

Flow:
1. User lands on commodity hub
2. Sees live price + forecast + linked reports
3. Reads multiple related reports
4. Starts to rely on repeated workflow
5. CTA shifts from content to capability:
   - unlock forecast depth
   - unlock scenario testing
   - unlock stock sensitivity
6. Pricing page emphasizes ongoing workflow value, not isolated article access
7. Checkout

### Path D: Pricing-intent visitor → proof → checkout
Best for:
- comparison visitors
- users already considering paying

Flow:
1. User lands on pricing page
2. Immediately sees who this is for and what decision workflow it replaces
3. Sees plan difference framed by outcome, not features
4. Sees one concrete example where Pro saves / makes money
5. Sees low-risk CTA
6. Checkout

## 7. Conversion Architecture by Page Type

## Homepage

### Role in funnel
- route visitors into the highest-intent workflow
- establish category and differentiation quickly

### What the homepage must do
1. answer what CommodityNode is in one sentence
2. answer who it is for
3. answer why it is different from news + charts
4. route the user into the best current setup

### Required sections

#### A. Hero
Headline should sell the core transformation.
Recommended direction:
- From raw commodity moves to stock and sector decisions.

Subheadline should describe the workflow.
Recommended direction:
- Track live commodity moves, see who wins or loses, compare model agreement, and stress-test scenarios before the street reprices.

Primary CTA:
- Start with today’s best setup

Secondary CTA:
- Open the Scenario Simulator

#### B. Today’s actionable setups
This should become the homepage’s highest-value block.

Each card contains:
- commodity + move
- one-line implication
- one-line “why this matters now”
- CTA

Recommended CTA language:
- Open the live setup
- See who it hits
- Model the move

#### C. Why CommodityNode instead of headlines
Need a comparison strip:
- Headlines tell you what moved
- Charts tell you where it traded
- CommodityNode tells you what it means for positions

#### D. Decision workflow explainer
Three steps only:
1. Start with the move
2. Translate the impact
3. Stress-test the scenario

#### E. Social proof / proof of seriousness
- live report count
- hubs tracked
- model stack named explicitly
- methodology/editorial transparency

### Homepage revenue objective
Push visitors into a real, timely setup where Pro becomes naturally relevant.

## Signal Report Pages

### Role in funnel
- satisfy search intent
- convert readers into workflow users

### Current strength
Report pages already do a good job of answering the query.

### What must improve
They need to create a stronger “next step ladder.”

### Ideal structure
1. Quick answer
2. Why this matters
3. Winners / losers
4. What to watch next
5. Live commodity hub CTA
6. Forecast CTA
7. Simulator CTA
8. Free signup / Pro CTA

### Required CTA sequence
Above the fold:
- Open the live hub

Mid-article:
- See the 90-day forecast context
- Model this move in the simulator

Bottom:
- Unlock stock-level and scenario workflow with Pro

### Revenue objective
The report should make the visitor feel informed but incomplete.
They understand the move, but they still need the hub / forecast / simulator to act.

---

## Commodity Hubs

### Role in funnel
- deepest free product surface
- strongest bridge to paid workflow

### What the hub should achieve
1. become the anchor page for that commodity
2. show live value
3. show repeated relevance
4. present the exact next premium capability

### Required hub flow
Top section:
- price
- daily change
- 30D / 90D forecast
- current narrative
- latest report freshness

Mid section:
- impact map / local universe
- related reports
- substitute / adjacent commodity comparisons

Commercial section:
- “Want to model a price shock scenario?”
- “Unlock stock-level sensitivity”
- “See full forecast workflow”

### Revenue objective
Turn commodity hubs into the strongest “aha” surface on the site.
This is where users should realize the product is not just content.

---

## Reports Archive

### Role in funnel
- habit / repeat-visit surface
- research shelf
- segmentation surface

### Current issue
The archive is useful, but can still feel like browsing.
Browsing is good for engagement but weaker for revenue.

### Needed changes
Add decision framing above the list:
- Best current setup
- Highest model disagreement
- Best pre-event read
- Editor’s pick for Pro workflow

Add commercial logic:
- after 2–3 archive interactions, push the simulator / pricing path harder
- highlight “best use with Pro” around forecast or company-sensitivity-related reads

### Revenue objective
Convert archive browsing into workflow adoption.

---

## Simulator

### Role in funnel
- strongest paid product surface
- highest purchase intent page on the site

### Why this page matters most
This is where users stop being readers and start behaving like buyers.

### Required structure
1. event-driven use case first
2. commodity selection second
3. forecast and agreement layer third
4. downstream winners / losers fourth
5. stock-level or deeper scenario output as paid unlock

### Commercial rule
The simulator should not say “pay to access.”
It should say:
- pay to finish the decision

### Ideal Pro boundary
Free can show:
- base forecast
- sector-first implications
- a subset of scenarios

Pro unlocks:
- company-level sensitivity
- save / compare scenarios
- richer shock combinations
- deeper export / workflow continuity

### Revenue objective
Make the simulator the most obvious reason to pay.

---

## Pricing Page

### Role in funnel
- not education only
- closing page

### Main job
Reduce hesitation.

### Current strongest angle
CommodityNode is cheap relative to one good decision.

### Pricing page should explicitly frame
Free:
- understand the move

Pro:
- act before repricing

Enterprise:
- operationalize across the desk

### Required sections
1. Who this is for
2. What workflow Pro unlocks
3. What you lose without it
4. Example ROI / avoided mistake / captured move
5. FAQ for trust reduction
6. low-friction CTA

### Pricing objective
The visitor should arrive already wanting Pro.
Pricing should remove remaining hesitation, not create new education work.

## 8. Revenue Triggers and Upgrade Moments

The site should trigger commercial asks only when intent is high.

### Trigger 1: Forecast curiosity
User wants:
- model agreement
- range depth
- confidence framing

Response:
- show preview
- CTA: Unlock the full forward view

### Trigger 2: Position translation
User wants:
- which stocks move
- company sensitivity
- downstream winners and losers

Response:
- CTA: Unlock stock-level decision workflow

### Trigger 3: Scenario anxiety
User wants:
- what happens if this extends / reverses / gets catalyzed

Response:
- CTA: Run the full scenario with Pro

### Trigger 4: Repeat behavior
User has read multiple reports or viewed multiple hubs

Response:
- CTA: Turn this into a repeatable workflow with Pro

## 9. Revenue-Maximizing CTAs

Avoid weak CTA styles like:
- Learn more
- View pricing
- Upgrade now

Preferred CTA language:
- Open the live setup
- See who this hits
- Check the 90-day outlook
- Model this move now
- Unlock stock-level sensitivity
- Finish the workflow with Pro
- Turn this setup into a repeatable decision process

These work better because they are job-based, not feature-based.

## 15. Final Strategic Rule

Every page should answer two questions before the visitor leaves:
1. What does this move mean?
2. What should I do next?

Revenue happens when the second question cannot be fully answered without Pro.

That is the commercial design principle CommodityNode should optimize around.
